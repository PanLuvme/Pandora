# Neo4j

Neo4j is a native graph database management system that stores data as nodes, relationships, and properties rather than in tables or documents. It is optimized for traversing highly connected data efficiently.

## Core Concepts

**Nodes** represent entities (e.g., Person, Product). **Relationships** connect nodes and always have a direction and a type (e.g., `[:KNOWS]`, `[:PURCHASED]`). Both nodes and relationships can hold **properties** as key-value pairs.

## Query Language

Neo4j uses **Cypher**, a declarative graph query language. A basic pattern match looks like:

```cypher
MATCH (a:Person)-[:KNOWS]->(b:Person)
WHERE a.name = "Alice"
RETURN b.name
```

## Key Strengths

- **Performance on connected data**: relationship traversal is O(1) per hop, unlike JOIN-heavy SQL queries that degrade with depth.
- **Schema flexibility**: nodes can have multiple labels; properties are optional and heterogeneous.
- **ACID compliance**: full transaction support.
- **Native graph storage**: relationships are stored as physical pointers, not index lookups.

## Common Use Cases

- Knowledge graphs and semantic networks
- Recommendation engines
- Fraud detection
- Identity and access management graphs
- Dependency and impact analysis

## Getting Started

```bash
# Run via Docker
docker run \
  --publish=7474:7474 --publish=7687:7687 \
  --env NEO4J_AUTH=neo4j/password \
  neo4j:latest
```

Access the browser UI at `http://localhost:7474`. The Bolt protocol endpoint is `bolt://localhost:7687`.
