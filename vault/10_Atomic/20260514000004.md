# Obsidian

Obsidian is a markdown-based note-taking application that uses **wikilinks** (`[[Note Title]]`) to create a personal knowledge graph stored entirely as plain text files on disk.

## Core Model

- **Notes** are individual `.md` files — no proprietary format, no database lock-in
- **Wikilinks** create bidirectional connections between notes, forming a traversable graph
- **Graph view** visualizes the link structure as an interactive node-edge diagram
- **Vault** is the root folder containing all notes — portable, version-controllable, and plain-text readable by any editor

## Key Strengths

- **Local-first**: all data lives on your filesystem; no cloud dependency
- **Plain text**: future-proof, grep-able, scriptable, and git-friendly
- **Extensible**: large plugin ecosystem (Dataview, Templater, etc.) for querying and automating the vault
- **Wikilink graph**: emergent structure — connections form naturally as you write

## Relationship to Knowledge Graphs

Obsidian implements a lightweight, human-curated knowledge graph at the filesystem level. Unlike a graph DB (e.g., Neo4j), there is no formal schema, query language, or relationship typing — structure is implicit in link co-occurrence and naming conventions. The two approaches are complementary: Obsidian for capture and authoring, a graph DB for structured reasoning and traversal.

## Common Use Cases

- Personal knowledge management (PKM) and second brain systems
- Zettelkasten and atomic note workflows
- Research note capture and synthesis
- Project and task tracking via linked notes
